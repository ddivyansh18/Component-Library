Program Structure In Javascript

Just like all programming languages, programs in JavaScript follow a definite syntax in structure and in this blog we will explore this structure.
This blog is based on the second chapter of book “Eloquent JavaScript” available here

Expressions and Statements

Any part of code which produces a value is called an expression.
Every written value is also an expression.
For ex : 25, “JavaScript”, true ;
A program is just a list of simple statements arranged to perform a complex task.
A semicolon after every statement is optional in JavaScript, but it is a good practice to use it after every statement to reduce ambiguity and confusion.

Bindings

Bindings or variables are used to catch and hold values.
In JavaScript, bindings are usually created using the keywords let, const and var.
For example : let one = 1, two = 2, three;
An empty binding like three in above statement, holds the value undefined.
We should avoid using var to create bindings as it has some confusing properties.
On the other hand, const is used to create constant bindings in which the variable can only hold a single value in its whole lifetime.
Reserved words called keywords can’t be used to create bindings.

Function

A function is like many statements wrapped in a single value.
This single value can be applied to execute the wrapped statements.
For example : console.log(“Hello World”) is a function that will print Hello World in the console.

Return Values

When a function produces a value using the wapped statements, it is said to return that value.
If a function writes text to screen, it is said to produce a side effect, not every function produces side effects.
For example : Math.max(1,5) returns value 5, to print this value to screen we need to execute console.log(Math.max(1,5)) which produces a side effect.

Control Flow

In JavaScript, statements are executed line by line from top to bottom.
We can also make some statements only execute based on certain conditions, this is called conditional execution. This is achieved using if and else statement.
The curly braces {} are used to group multiple statements into a single statement. We can also chain multiple if-else statements if required.

While and do-while loops

A loop allows us to do repetitive work easily using some simple statements.
A statement starting with while indicates a while loop, which runs “while a condition is true”.
Do while loops on the other hand run atleast once, and are “exit controlled loops”, which means that the loop condition is checked after execution of every iteration.

Indenting code

Indentation means adding extra spaces, tabs and empty lines in the code to make it more readable.
It is a good practice to indent our code so that other people can easily read it.

For loop

It is also a type of loop just like while and do while, but the difference here is that a counter like variable to track the progress of the loop.
Also the syntax contains mandatory two semicolons. “break;” is used to instantly come out of a loop

Switch

Instead of a if else ladder sometimes it is easier to use switch-case statement.
There are multiple case statements inside a switch statement, which can be executed based on the case label, if no case label matches then the default case is used.
It is required to use “break;” statement if we want only code related to that label to execute, in case break statement is not used, the execution goes on until a break statement is encountered.

Comments

Comments convey additional information to someone who is reading that code, comments are not executed at all, in JavaScript // is used for single line comment and /**/ is used for multiline comment.

Thus, we discussed a lot of programming constructs which are building blocks of every JavaScript program.
